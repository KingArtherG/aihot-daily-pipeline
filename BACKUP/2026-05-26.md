## [Harness、Scaffold 与 AI 智能体术语辨析](https://huggingface.co/blog/agent-glossary) `#18`

> Hugging Face博客厘清AI智能体关键术语：模型是核心，Scaffolding定义行为，Harness执行循环，Agent = Model + Harness。

Hugging Face官方博客发布文章，厘清AI智能体领域易混淆的术语。文章指出：模型（如Claude、GPT）本身是无记忆、无循环的LLM。其行为由Scaffolding（行为定义层，如系统提示、工具描述）塑造，而Harness（执行层）负责调用模型、处理工具调用与控制循环，是智能体运行的核心。两者结合，模型才能成为智能体。文章以Claude Code、Codex为例说明同一模型搭配不同Harness产生迥异体验。

来源：Hugging Face Blog。

核心定义：Model = 无记忆/循环的LLM；Scaffolding = 行为定义（提示、工具描述）；Harness = 执行层（调用模型、工具、循环）。

Agent = Model + Harness。

示例：Claude Code与Codex使用相同底层模型但Harness不同导致体验差异。

术语标准化有助于开发者理解智能体架构，避免概念混淆，对构建可靠AI Agent应用至关重要。

帮助社区形成统一的智能体架构讨论框架，可能影响未来开源Agent框架设计。
